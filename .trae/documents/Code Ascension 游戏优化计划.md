# 优化 Code Ascension 游戏体验

我将根据您的要求，对游戏进行以下全方位的优化。

## 1. 视觉与动画系统重构
- **右侧出牌区域**：在界面中部的 "VS" 区域右侧新增一个专属的 `played-cards-container`。
- **非线性平滑动画**：
  - 实现 `animateCardPlay(card, sourceElement)` 函数。
  - 玩家/AI 出牌时，卡牌将从手牌位置平滑飞向右侧出牌区。
  - 使用 CSS `cubic-bezier` 实现非线性运动（快出慢停）。
  - 出牌区采用 **从左往右** 排列，新牌覆盖旧牌（`margin-left: -x`），旧牌自动降低透明度。

## 2. 核心逻辑修正：普通杀限制
- **修复无限出杀 Bug**：
  - 在 `Game.playCard` 中增加强制检查逻辑。
  - 判定条件：`if (card.actionType === 'ATTACK' && player.flags.attackCount >= 1 && !player.hasEquip('equip_multi_attack'))` 则阻止出牌并提示。
  - 同时修正 AI 逻辑，确保 AI 不会试图打出无效的第二张杀。

## 3. UI 美化：自绘 SVG 图标
- **建立 SVG 图标库**：为每种卡牌（杀、闪、桃、锦囊等）和角色（创业者、资本家）手绘简约风格的 SVG 图标。
- **卡牌显示升级**：
  - 手牌：在卡牌中心显示大图标。
  - 装备区：重写装备显示逻辑，不再只显示文字，而是显示“图标 + 名称”的精简卡牌样式。

## 4. 锦囊“备用方案” (Satchel) 交互重做
- **新交互设计**：
  - 在玩家装备区旁新增一个永久显示的 **[备用方案]** 槽位。
  - **存入**：当槽位为空且装备了“备用方案”时，点击手牌会弹出“存入备用方案”选项。
  - **取出**：当槽位有牌时，点击该槽位可选择“取出/使用”。
- **AI 适配**：AI 也会利用此逻辑进行存取操作。
